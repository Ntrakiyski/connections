# Code Gate Prompts

Use only concerns supported by the completed task, feature artifacts, project
instructions, and retained evidence.

- Does every changed file serve an approved task or acceptance criterion?
- Does the change preserve relevant project boundaries and conventions?
- Do seam-level tests or direct checks prove the affected behavior?
- Are error, empty, loading, and permission states considered where the change
  makes them relevant?
- Does each claimed result link to retained evidence?
