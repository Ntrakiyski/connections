# Gate Checklist Methodology

These references prompt a gate; they are not a checklist to copy. Every check
must name the project artifact or evidence that proves it.

1. Read the active artifact, project instructions, `.autonomous/state.json`,
   phase manifest, and relevant evidence.
2. Select only the matching mode reference.
3. Turn each relevant concern into a specific yes-or-no check for this work.
4. Link each pass to a source path, command result, or evidence file.
5. Record missing evidence as `fail` or `blocked`, never as a pass.

Write the result to `.autonomous/phases/gate-<mode>.md`. The gate result must
agree with the phase manifest and active milestone state.
