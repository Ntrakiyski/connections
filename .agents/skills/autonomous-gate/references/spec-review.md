# Specification Gate Prompts

Use only concerns supported by the active Spec Kit feature and project
documents.

- Do `spec.md`, `plan.md`, and `tasks.md` describe the same bounded feature?
- Does the plan name the actual seams, data, integrations, and constraints?
- Is each requirement represented by an ordered implementation or verification
  task?
- Can a reviewer execute the stated acceptance checks without invention?
- Are unresolved decisions explicitly blocked rather than hidden?
